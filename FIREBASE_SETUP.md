# Firebase Setup Guide

This guide will help you complete the Firebase Authentication and Leaderboard setup for Smart Health app.

## Prerequisites

- Firebase project already created (smart-health-dc9e4)
- `google-services.json` already added to the app

## Setup Steps

### 1. Configure Google Sign-In in Firebase Console

1. Go to [Firebase Console](https://console.firebase.google.com/)
2. Select your project: **smart-health-dc9e4**
3. Navigate to **Authentication** > **Sign-in method**
4. Click on **Google** provider
5. **Enable** Google Sign-In
6. Save the configuration

### 2. Add SHA-1 Certificate Fingerprint

Google Sign-In requires your app's SHA-1 certificate fingerprint.

#### Get Debug SHA-1 (for development):

Open terminal in your project directory and run:

```bash
cd android
./gradlew signingReport
```

Or use keytool directly:

```bash
keytool -list -v -keystore ~/.android/debug.keystore -alias androiddebugkey -storepass android -keypass android
```

For Windows:
```bash
keytool -list -v -keystore %USERPROFILE%\.android\debug.keystore -alias androiddebugkey -storepass android -keypass android
```

#### Get Release SHA-1 (for production):

Use your release keystore:

```bash
keytool -list -v -keystore /path/to/your/release.keystore -alias your-alias-name
```

#### Add SHA-1 to Firebase:

1. In Firebase Console, go to **Project Settings** (gear icon)
2. Scroll down to **Your apps** section
3. Click on your Android app
4. Click **Add fingerprint**
5. Paste your SHA-1 certificate fingerprint
6. Click **Save**

### 3. Download Updated google-services.json

After adding your SHA-1 fingerprint:

1. In Firebase Console > **Project Settings** > **Your apps**
2. Click on **google-services.json** download button
3. Replace the existing `app/google-services.json` with the new one
4. The new file will contain OAuth client configuration

### 4. Update strings.xml with OAuth Client ID

After downloading the updated google-services.json:

1. Open the new `google-services.json` file
2. Find the `oauth_client` array with `client_type: 3` (Web client)
3. Copy the `client_id` value
4. Open `app/src/main/res/values/strings.xml`
5. Replace `YOUR_OAUTH_CLIENT_ID_HERE` with your actual client ID

Example:
```xml
<string name="default_web_client_id">123456789-abcdefghijklmnopqrs.apps.googleusercontent.com</string>
```

**Note:** The Web OAuth client ID (client_type: 3) is automatically generated by Firebase when you add your SHA-1 fingerprint.

### 5. Create Firestore Database

1. In Firebase Console, go to **Firestore Database**
2. Click **Create database**
3. Choose **Start in test mode** (for development)
4. Click **Next**
5. Select your preferred location
6. Click **Enable**

### 6. Configure Firestore Security Rules

Set up basic security rules for development:

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Users collection - users can read all, but only write their own data
    match /users/{userId} {
      allow read: if true;
      allow write: if request.auth != null && request.auth.uid == userId;
    }
  }
}
```

For production, update rules to be more restrictive:

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId} {
      allow read: if request.auth != null;
      allow write: if request.auth != null && request.auth.uid == userId;
    }
  }
}
```

### 7. Test the Setup

1. Build and run the app
2. The app should now open with the Sign-In screen
3. Click **Sign in with Google**
4. Complete the Google Sign-In flow
5. After successful sign-in, you should be redirected to the Home screen
6. Navigate to the Leaderboard to see global rankings

## Troubleshooting

### Sign-In Fails

**Error:** `ApiException: 10: Developer error`
- Cause: SHA-1 not configured or mismatch
- Solution: Add correct SHA-1 fingerprint to Firebase Console and download new google-services.json

**Error:** `default_web_client_id not found`
- Cause: strings.xml not updated with OAuth client ID
- Solution: Follow Step 4 above to add the correct client ID

### Leaderboard Empty

- Cause: No users in Firestore yet
- Solution: Sign in with multiple accounts to populate the leaderboard

### Points Not Syncing

- Cause: User not signed in
- Solution: Ensure user completes Google Sign-In before earning points
